# エルメッセージフォーム 運用マニュアル

## 📌 基本情報

**制限**: 無料プランでは**最大3フォーム**まで

**特徴**:
- ✅ デフォルトで自動返信機能あり
- ✅ 友だち追加不要
- ✅ セットアップが簡単

---

## 🎯 推奨：3フォーム運用戦略

### 戦略1: イベント種類別割り当て

| フォーム番号 | 用途 | 想定使用頻度 |
|------------|------|------------|
| **フォーム1** | 定例会議・役員会 | 月1回 |
| **フォーム2** | イベント・行事 | 月2-3回 |
| **フォーム3** | アンケート・調査 | 不定期 |

### 戦略2: 時期別ローテーション

**年間計画例**:

```
1-4月 : フォーム1で第1四半期イベント管理
5-8月 : フォーム2で第2四半期イベント管理
9-12月: フォーム3で第3-4四半期イベント管理
```

**メリット**: 
- 同時に複数イベントを管理しない場合に有効
- 各フォームの回答データは保持されるため、過去イベントも参照可能

## 🔄 フォーム再利用の手順

### ステップ1: データのバックアップ

**重要**: フォームを再編集する前に、必ず回答データをバックアップ

1. エルメッセージ管理画面でフォームを開く
2. 回答データをCSVでダウンロード
3. Googleスプレッドシートなどに保存

### ステップ2: フォーム内容の変更

1. フォーム編集画面を開く
2. フォーム名を変更
3. 日時などの可変項目を更新
4. 固定項目（出欠選択肢など）はそのまま

### ステップ3: 新規配信

1. LINEで新しいフォームリンクを配信
2. 配信文に「〇〇の出欠確認です」と明記

## 💡 運用Tips

### Tip 1: 固定項目と可変項目を分ける

**固定項目**（毎回変更しない）:
- 名前（竹号）
- 出欠選択肢
- コメント欄

**可変項目**（都度変更）:
- イベント名
- 日時
- 場所

### Tip 2: 命名規則を統一

フォーム名は以下の形式で統一：
```
YYYY/MM/DD 〇〇 出欠確認
```

例:
- `2026/01/15 新年会 出欠確認`
- `2026/03/20 総会 出欠確認`

### Tip 3: 過去データの整理

**月次作業**:
- 前月のフォーム回答をCSVダウンロード
- Googleスプレッドシートに統合
- フォームを次回イベント用に再編集

**推奨フォルダ構造**:
```
Googleドライブ/エルメッセージフォーム回答/
├── 2026年1月_新年会.csv
├── 2026年2月_定例会議.csv
├── 2026年3月_総会.csv
└── ...
```

## 🆚 フォーム使い分けマトリクス

### いつ、どのフォームを使うべきか

| 状況 | 推奨フォーム | 理由 |
|------|-----------|------|
| 月1回の定例会議 | フォーム1固定 | 同じフォーマットで統一性 |
| 不定期イベント | フォーム2 | 柔軟に内容変更可能 |
| 年1-2回のアンケート | フォーム3 | 出欠以外の質問項目 |
| 緊急時 | フォーム3 | 常に空けておく |

## ⚠️ 注意事項

### やってはいけないこと

❌ **同時に2つのイベントで同じフォームを使う**
- 回答が混ざってしまう
- どちらのイベントの回答か区別できない

❌ **バックアップせずにフォームを編集**
- 過去の回答データが失われる可能性

❌ **複雑すぎる質問項目**
- 回答率が下がる
- シンプルな構成を心がける

### ベストプラクティス

✅ **イベント終了後すぐにデータをダウンロード**
✅ **フォーム名に日付を含める**
✅ **3フォームの役割を明確に決める**
✅ **年間スケジュールを立ててフォーム配分を計画**

## 📊 効果測定

### 追跡すべき指標

- **回答率**: 送信数 ÷ 回答数
- **回答時間**: 配信から回答完了までの平均時間
- **フォーム編集頻度**: 月何回再利用しているか

### 改善ポイント

| 課題 | 対策 |
|------|------|
| 回答率が低い | 配信文を工夫、締切を明記 |
| 回答に時間がかかる | 項目を減らす、必須項目を最小限に |
| フォーム数が足りない | 有料プランの検討、イベント統合 |

## 🔧 トラブルシューティング

### Q1: フォームが3つでは足りない

**解決策**:
1. イベントの優先順位をつけ、重要度の低いものはスプレッドシート共有で代用
2. 複数イベントを1つのフォームに統合（質問で「どのイベントですか？」を追加）
3. 有料プランを検討

### Q2: 過去のデータを確認したい

**解決策**:
- 定期的にCSVダウンロードしてGoogleドライブに保存
- スプレッドシートで一元管理

### Q3: 自動返信メッセージをカスタマイズしたい

**制限**: エルメッセージの無料プランでは詳細なカスタマイズは不可

**代替案**:
- 有料プランにアップグレード
- Google Apps Scriptで独自実装（友だち追加必須）
